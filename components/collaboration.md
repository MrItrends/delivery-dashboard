# Collaboration Components

## 21 — Activity Feed

### Purpose

Communicate what has changed on an object over time. The Activity Feed is an immutable, append-only record of everything that happened — presented as a readable timeline.

### Each Feed Item Contains

| Field | Content |
|-------|---------|
| Actor | Avatar + name of who made the change |
| Action | What they did — in plain English |
| Object | What object was affected (if different from current context) |
| Time | Relative time ("4 minutes ago") with absolute tooltip |
| Optional Metadata | Previous value → new value for field changes |

### Example Entries

```
[Ahmed] marked [Draft curriculum materials] as Complete · 4 min ago
[Sarah] uploaded [Q3 Progress Plan.pdf] to Evidence · 12 min ago
[Marcus] changed due date from 14 Sept → 21 Sept · 2 hours ago
[System] Milestone 2 became At Risk due to dependency · 3 hours ago
```

### Rules

- **Immutable** — feed items cannot be edited or deleted
- **Append-only** — new items appear at the top
- Feed entries are **generated by the system** based on changes; they are not manually created
- **Never use the Activity Feed as a chat or message thread** — that is the Comment Thread's job
- Feed items that reference other objects are clickable links

### Design

- Feed uses a vertical timeline line connecting items
- Actor avatars are 24px — smaller than comment avatars (32px) to signal the feed is secondary information
- Timestamps are neutral-400 (de-emphasized)
- Metadata changes show: `Previous value → New value` in a monospace, dimmed style

## 22 — Comment Thread

### Purpose

Contextual discussion attached to a specific object. Comments are where humans communicate about delivery — not a separate collaboration tool, not a chat feature, but a conversation embedded in the thing it's about.

### Anatomy (Single Comment)

```
[Author Avatar 32px]  [Author Name]  [Timestamp]  [···]
  Comment text — supports @mentions, inline links
  
  [Attached files]
  
  [Reply ▾]  [React 👍]  [Resolve]
```

### Thread Structure

```
Parent Comment
  └─ Reply
  └─ Reply
Parent Comment (resolved) ← collapsed by default
Parent Comment
```

### Features

| Feature | Behavior |
|---------|---------|
| @Mentions | Type `@` to search team members; notifies them |
| Attachments | Drag or click to attach files to a specific comment |
| Reactions | Emoji reactions (👍 ✓ 👀) — no comment required |
| Resolve | Marks a thread as addressed — collapses it |
| Edit | Author can edit their own comment (edit shows "(edited)") |
| Delete | Author can delete their own comment (leaves "Comment deleted" placeholder) |
| Replies | Threaded replies up to one level (replies to replies flatten into thread) |

### Rules

- Comments belong to **objects** — not to users, not to pages
- A comment on an Intervention is visible on that Intervention's Discussion tab
- Comments are **never standalone** — they always have a parent object
- Resolved threads are **hidden by default** but accessible via "Show resolved (n)"

### Rich Text Support

Comments support: bold, italic, inline code, links, bullet lists, numbered lists. No full markdown — keep it simple.

## 23 — File Preview

### Purpose

View a file without downloading it. File Preview provides an in-browser reading and annotation experience tied to the file's context in the system.

### Supported Preview Types

| Type | Preview |
|------|---------|
| PDF | Full PDF viewer with page navigation, zoom |
| Images (JPG, PNG, GIF, WebP) | Full-resolution with zoom |
| Word (.docx) | Rendered HTML preview |
| Excel (.xlsx) | Tabular preview, read-only |
| PowerPoint (.pptx) | Slide-by-slide preview |
| Video (MP4, MOV) | In-browser player |
| CSV | Tabular preview, sortable columns |
| Other | "Preview not available — [Download]" |

### Anatomy

```
┌──────────────────────────────────────────────────────┐
│ Q3 Progress Plan.pdf  ·  v3  ·  Ahmed  ·  15 Sept   │
│                                         [↓] [Share] │
├───────────────────────────────┬──────────────────────┤
│                               │ Versions             │
│  [PDF Preview]                │ v3 — Current         │
│                               │ v2 — 12 Sept         │
│  Page 1 of 12  < >           │ v1 — 1 Sept          │
│                               │──────────────────────│
│                               │ Comments (3)         │
│                               │ [Thread]             │
└───────────────────────────────┴──────────────────────┘
```

### File Preview Opens In

- Inspector panel (default) — when clicking a file in a Files tab
- Full-screen overlay — when user clicks "Expand" or opens from search
- Within Milestone approval panel — evidence file alongside approval form

## 24 — Avatar Stack

### Purpose

Represent collaboration and presence compactly. Shows who is working on or has access to an object.

### Specifications

```
Single avatar:        32px circle, border: 2px white
Stacked avatars:      Each overlaps by 8px to the left
Maximum visible:      5 avatars
Overflow:             +12 badge (neutral-700 background, white text)
Hover:                Tooltip shows full name list
```

### States

| State | Visual |
|-------|--------|
| Online | Green dot (8px) bottom-right |
| Away | Yellow dot |
| Offline | No dot |
| Viewing | Blue ring pulse (real-time presence) |

### Sizing

| Context | Size |
|---------|------|
| Header / metadata | 24px |
| Toolbar / Inspector | 28px |
| Comment author | 32px |
| Profile references | 40px |

### Rules

- Avatar initials (2 letters) when photo is not available
- Background color is deterministically assigned from the user's ID — consistent across sessions
- Never shows more than 5 avatars before collapsing to overflow
