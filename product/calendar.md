# Calendar

## Purpose

The Calendar provides a **unified temporal view** across all delivery objects: Activities, Milestones, Reviews, Deadlines, and Events — all in one place.

---

## Business Goal

Allow delivery teams to understand what is happening across the programme this week, this month, and this quarter — without opening individual projects.

---

## What the Calendar Displays

| Object | Appears As |
|--------|-----------|
| Activity due dates | Task bar or point |
| Milestones | Diamond marker |
| Reviews | Meeting event |
| Budget deadlines | Financial event |
| Reports due | Report event |
| External events | Custom event |

---

## Calendar Views

| View | Purpose |
|------|---------|
| Day | Detailed today view |
| Week | Current week overview |
| Month | Standard monthly view |
| Quarter | 13-week delivery overview |
| Agenda | Chronological list view |

---

## Filters

Users can filter by: Object type, Priority Area, Project, Intervention, Owner, Team, Status, Tags.

A Project Manager can view their Project's milestones only. A delivery lead can view all Activities assigned to their team.

---

## Calendar + Activities

Activity due dates flow into the Calendar automatically. No duplication. The Calendar reads from the same database.

---

## Deadlines vs. Events

| Type | Description |
|------|-------------|
| Deadline | Hard due date from an Activity or Milestone |
| Review | Scheduled meeting or oversight session |
| Approval Window | Budget or report approval deadline |
| Custom Event | Manually created event |

---

## Quick Create

Users can create Activities, Milestones, or Events directly from the Calendar by clicking a date. Context is preserved — the object defaults to the correct project.

---

## Conflict Detection

When Activities are overloaded on a single date, the Calendar surfaces this visually. Team capacity risk becomes visible before it becomes a delivery problem.

---

## External Calendar Integration

Sync to: Google Calendar, Outlook / Microsoft 365, Apple Calendar, iCal (export).

Events created in the Delivery Dashboard appear in personal calendars. External events do not automatically import.

---

## Claude Implementation Notes

The Calendar should feel like a **delivery timeline**, not a meeting scheduler. Design it with the Milestone timeline as a primary design reference — not Google Calendar.

Key visual priority: surfacing **deadlines and milestones** over meetings and events. This is a delivery calendar, not a personal scheduling tool.
