# Milestones

## Purpose

Milestones are **formal delivery checkpoints**. They mark the moments when a measurable outcome must be achieved. They are not tasks — they are **gates**.

When a Milestone is reached, it must be evidenced, reviewed, and approved.

---

## Business Goal

Create a formal accountability mechanism that ensures government delivery programmes have clear, measurable waypoints — not simply progress bars.

---

## Core Philosophy

A Milestone is not a task with a due date. A Milestone is a **confirmed outcome**.

- Tasks are completed. Milestones are **achieved and verified**.
- Tasks may be re-opened. Milestones, once approved, are permanent.
- Tasks have owners. Milestones have owners, reviewers, and approvers.

## Milestone Status

| Status | Description |
|--------|-------------|
| Draft | Being defined |
| Planned | Approved for delivery |
| Active | Work underway |
| Ready for Review | Outcome claimed, awaiting approval |
| Approved | Formally confirmed |
| Achieved | Verified complete |
| Delayed | Missed target date |
| Blocked | Cannot proceed |
| Archived | Closed permanently |

---

## Evidence Requirements

A Milestone cannot be marked Achieved without evidence:

| Type | Examples |
|------|---------|
| Documents | Reports, Contracts, Policies |
| Data | KPI results, Survey data, System data |
| Media | Photos, Videos, Recordings |
| Approvals | Sign-offs, Certificates |
| URL | External verification |

Evidence is permanently attached. Cannot be deleted.

---

## Milestone Lifecycle

```
Draft → Planned → Active → Ready for Review → Approved → Achieved → Archived
```

Milestones can become Delayed or Blocked from any active state.

---

## Approval Flow

```
Owner claims completion → Reviewer assesses evidence → Approver gives formal approval
```

Every step is logged: Who, When, Decision, Comments.

## Claude Implementation Notes

Milestones should feel **significant**. Completing one should feel different from completing a task.

Design the Milestone as a formal evidence workspace with a clear approval workflow — not just a checklist item.
