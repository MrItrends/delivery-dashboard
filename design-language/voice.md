# Voice & Copywriting

## The Three Words

**Short. Professional. Human.**

Every word in the product — button labels, error messages, empty states, tooltips, notifications, onboarding prompts — must satisfy these three qualities simultaneously.

---

## What to Avoid

### Technical Language
Do not expose system language to users. Never write what the software is doing — write what it means.

✗ "Error 422: Unprocessable Entity"  
✓ "Check the highlighted fields and try again."

✗ "Foreign key constraint violation"  
✓ "This intervention cannot be deleted while it has active activities."

### Marketing Language
The product is not a landing page. Enthusiasm and superlatives are inappropriate.

✗ "Unleash the power of government delivery!"  
✗ "You're crushing it!"  
✗ "Welcome to your delivery journey."  
✓ "Welcome to the Delivery Dashboard."

### AI Language
Do not write like AI-generated copy sounds. Over-structured, hedging, padding.

✗ "It appears that the due date may potentially have passed."  
✓ "This activity is overdue."

✗ "Oops! Looks like something went wrong on our end. Don't worry!"  
✓ "Unable to save. Please try again."

---

## The Voice

The Delivery Dashboard speaks like:

**A senior civil servant who knows what they're doing** — not a colleague who's apologizing, not a chatbot trying to be helpful, not a startup that's excited about everything.

Confident. Direct. Helpful. Brief.

---

## Button Labels

Buttons should say what they do. Not vaguely confirm.

| ✗ Avoid | ✓ Prefer |
|---------|---------|
| OK | Got it |
| Submit | Save Activity |
| Confirm | Archive |
| Yes | Delete |
| Cancel | Discard Changes |
| Click here | View Intervention |

**The button label should complete the sentence:** "I want to ___."

---

## Error Messages

Every error message explains: **What happened. Why. What to do.**

```
✗ "An error occurred. Please try again later."

✓ "Unable to save this activity.
   The due date must be after the start date.
   [Update Due Date]"
```

```
✗ "You don't have permission."

✓ "You don't have access to this intervention.
   Contact your workspace administrator to request access.
   [Request Access]"
```

---

## Empty State Copy

Empty states teach what a section is for. They do not apologize for being empty.

```
✗ "No data to display."
✗ "You haven't created any activities yet."

✓ "No activities yet.
   Activities are the individual pieces of work that move
   an intervention forward. Create your first to get started.
   [Create Activity]"
```

---

## Notification Copy

Notifications are action requests. Every notification must clearly state what happened and what (if anything) should be done.

```
✗ "You have a new notification."

✓ "Ahmed Yusuf assigned you to 'Submit Planning Application'.
   Due 15 September."
   [View Activity]

✓ "Healthcare Milestone is 3 days overdue.
   The completion date was 20 August."
   [Review Milestone]
```

---

## Confirmation Messages

Brief. Confident. Undo offered where appropriate.

```
✓ "Activity archived."   [Undo]
✓ "Budget updated."
✓ "Report generated."    [View Report]
✓ "Milestone approved."
```

Never: "Your action was successful!" / "Done!" / "Great job!"

---

## Status Labels

Status labels are nouns or past-tense verbs. Never questions or sentences.

```
✓ "Not Started"
✓ "In Progress"
✓ "Blocked"
✓ "Completed"
✓ "At Risk"
✓ "Archived"

✗ "This is currently in progress"
✗ "Blocking!"
✗ "DONE ✓"
```

---

## Formatting Rules

**Capitalization:** Title case for headings, buttons, navigation. Sentence case for everything else.

**Punctuation:** No exclamation marks (ever). No ellipsis in labels. Full stops in multi-sentence descriptions, none in single-line labels.

**Numbers:** Use numerals always (3, not three). Use commas for thousands (1,250). Use % not "percent."

**Dates:** Consistent. Always "15 Sept 2024" or "15 September 2024" — never "Sept 15" or "9/15/2024."

---

## The Voice Test

Copy passes when:

1. A non-technical government official reads it and immediately understands
2. A senior civil servant reads it and does not feel talked down to
3. It cannot be made shorter without losing meaning
4. It contains no marketing language, AI-speak, or technical jargon
5. The tone feels appropriate for a consequential government context
