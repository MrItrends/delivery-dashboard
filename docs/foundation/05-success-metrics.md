# 05 — Success Metrics

The Delivery Dashboard exists to improve delivery outcomes rather than simply digitise reporting.

Success should be measured across five dimensions.

---

## 1. Product Adoption

Measure how successfully organisations adopt the platform.

| Metric | Description |
|--------|-------------|
| Active workspaces | Number of live operating environments |
| Weekly active users | Users active in any 7-day period |
| Daily active users | Users active each day |
| Monthly retention | Users returning month-over-month |
| Cross-ministry adoption | Spread across government entities |
| Invitation acceptance rate | % of invited users who onboard |

---

## 2. Delivery Performance

Measure improvements in execution.

| Metric | Description |
|--------|-------------|
| Activity completion rate | % of activities completed on time |
| Milestone completion rate | % of milestones achieved on schedule |
| Average project completion | % completion across active projects |
| Overdue activities | % of activities past due date |
| Time to resolve blockers | Average days to clear a blocker |
| Decision turnaround time | Time from request to decision |

---

## 3. Collaboration

Measure how teams work together.

| Metric | Description |
|--------|-------------|
| Comments created | Total contextual discussion volume |
| Mentions used | @ references per period |
| Assignments completed | Accepted and finished assignments |
| Shared ownership | Objects with multiple contributors |
| Review approvals | Approvals processed per period |
| Cross-team participation | Users contributing outside their team |

The platform should encourage collaborative behaviour rather than isolated reporting.

---

## 4. Operational Efficiency

Measure reductions in administrative overhead.

| Metric | Description |
|--------|-------------|
| Reduction in spreadsheet usage | Self-reported or inferred |
| Reduction in duplicate reporting | Tracked through user surveys |
| Faster report generation | Time from request to delivery |
| Time saved on executive updates | Estimated hours saved weekly |
| Automated reminders | Notifications triggered without manual action |
| Calendar integrations used | External calendar sync adoption |

---

## 5. Product Quality

Monitor the health of the platform itself.

| Metric | Target |
|--------|--------|
| Search success rate | > 95% |
| Average page load time | < 2 seconds |
| Error rate | < 0.1% |
| Offline sync success | > 99% |
| Accessibility compliance | WCAG AA minimum |
| User satisfaction score | > 4.2 / 5.0 |

---

## Success Statement

> The product succeeds when government teams spend **less time collecting information** and **more time delivering outcomes**.
