# 29 — Team Management

## Purpose

The platform manages delivery through **people**. Therefore, people should never feel like secondary objects.

Team Management provides visibility into organizational structure, capacity, responsibility and collaboration.

---

## Team Object

Each Team contains:

| Field | Description |
|-------|-------------|
| Name | Team name |
| Description | Purpose and scope |
| Department | Organizational unit |
| Lead | Team leader |
| Members | All members |
| Responsibilities | What this team owns |
| Projects | Associated projects |
| Interventions | Active delivery work |
| Availability | Capacity status |
| Performance | Delivery metrics |

---

## Member Profile

Every user has a profile containing:

| Field | Description |
|-------|-------------|
| Role | System role |
| Department | Organizational unit |
| Manager | Reporting line |
| Skills | Competency areas |
| Assignments | Active activities |
| Availability | Capacity status |
| Recent Activity | Latest contributions |
| Current Workload | Load indicator |
| Contact Information | How to reach them |

---

## Workload

Visualize:
- Assigned Activities
- Overdue Work
- Upcoming Reviews
- Capacity
- Availability

The goal is **balanced delivery**, not simply maximum utilization.

---

## Organizational Structure

Support:
- Ministries
- Departments
- Delivery Units
- Cross-functional Teams
- Task Forces
- Partner Organizations

The structure should remain flexible.

---

## Team Views

| View | Purpose |
|------|---------|
| Directory | Full member list with search |
| Org Chart | Organizational hierarchy |
| Capacity | Workload distribution |
| Assignments | What everyone is working on |
| Performance | Delivery metrics |
| Availability | Who is available |

---

## Collaboration Insights

Display:
- Recent Collaborators
- Frequently Worked Together
- Current Reviews
- Shared Projects

This helps build stronger delivery teams.

---

## Permissions

| Role | Access |
|------|--------|
| Team Lead | Manage Team, Assign Work, Approve Reviews, View Capacity |
| Members | View Assigned Work, Update Activities, Comment, Collaborate |
| Observers | Read Only |

---

## Claude Implementation Notes

People are not resources. **People are collaborators.**

Design Team Management around **visibility, trust and accountability** rather than administration.

---

## Review Checklist

- [ ] Organizational structure is clear
- [ ] Workload is immediately visible
- [ ] Team capacity shown accurately
- [ ] Assignment transparency maintained
- [ ] Collaboration insights surface naturally
- [ ] Profiles feel human, not administrative
