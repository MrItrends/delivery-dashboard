# Navigation System

## Philosophy

Navigation should **disappear**. Users should stop thinking about navigation after the first day. The interface should always make the next action obvious.

---

## Primary Navigation

Located on the left. Persistent. Minimal. Always visible. Contains only major destinations.

```
Workspace

Portfolio

Performance

Reports

Calendar

──────────

Team

Notifications

Settings
```

No icons beside every item. No coloured indicators. No decorative elements.

**Typography and spacing carry the hierarchy.**

---

## Secondary Navigation

Appears inside a selected object. Changes with context.

**Example — Project:**
```
Overview
Activities
Milestones
Budget
KPIs
Files
Team
History
Settings
```

---

## Breadcrumbs

Every page displays its position in the hierarchy. Breadcrumbs are interactive — each level is clickable.

```
Workspace / Portfolio / Healthcare / National Hospital Upgrade / Activities
```

---

## Command Palette

The Command Palette is a **first-class navigation tool**.

**Shortcut:** `⌘K` / `Ctrl+K`

Users can navigate, search, create, assign, invite, export, and generate reports — **without touching the sidebar**.

Inspired by: Linear, Raycast, Figma.

---

## Universal Search

Search returns: Projects, Activities, Users, Reports, Comments, Files, Budgets, Milestones, Interventions.

Results appear instantly. Search is workspace-wide and permission-aware.

---

## Right Inspector

Instead of navigating away, details open inside an inspector panel. This preserves context.

```
Click Activity → Right Panel Opens → Background remains visible
```

---

## Navigation Memory

Navigation remembers:
- Last workspace
- Recent projects
- Recent reports
- Pinned objects
- Saved filters
- Recent searches

The interface adapts to the user's workflow.

---

## Mobile Navigation

The mobile experience prioritises:
- Today's work
- Assigned activities
- Notifications
- Quick updates
- Search

The full desktop hierarchy remains available through drawers and bottom sheets.

---

## Navigation Design Test

The user should always know:
1. Where am I?
2. What am I looking at?
3. What can I do next?
4. How do I get back?

If these four questions cannot be answered without hesitation, the navigation system has failed.
